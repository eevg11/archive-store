# ARCHIVE // Conceptual E-Commerce Store

A sleek, minimalist, and fully interactive single-page e-commerce store built for a conceptual clothing brand. This project demonstrates modern responsive layouts, data-driven UI generation, and dynamic shopping cart logic using vanilla web technologies.

## 🚀 Live Demo
👉 [Launch Live Project](https://eevg11.github.io/archive-store/)

## ✨ Core Features
* **Dynamic Catalog Rendering:** Products are generated dynamically from a JavaScript data structure (simulating a backend API).
* **Advanced Filtering & Sorting:** Instant product filtering by categories and real-time sorting by price (Low to High / High to Low) without page reloads.
* **Fully Functional Cart:** Interactive slide-out shopping cart sidebar that handles adding items, computing quantities, calculating total price updates, and removing items instantly.
* **Flawless Grid Responsiveness:** Custom CSS Grid implementation with layout safety guards to prevent image stretching when viewing single items.
* **Smooth Navigation:** Native anchor scroll-behavior configuration for fluid page section transitions.

## 🛠️ Tech Stack
* **HTML5:** Semantic architecture (`<header>`, `<section>`, `<nav>`, `<footer>`).
* **CSS3:** Custom variables, Flexbox, CSS Grid (`auto-fill` mechanics), and `backdrop-filter: blur` effects.
* **JavaScript (ES6+):** Vanilla DOM manipulation, conditional filtering (`.filter()`), array sorting (`.sort()`), and object state tracking.

##⚠️ Technical Note: Media Assets
*For demonstration purposes and to keep the repository lightweight, this project utilizes direct external image links from Unsplash via production-ready URLs.

*Advantage: The live demo on GitHub Pages works out of the box instantly without heavy image files cluttering the source tree.

*Production Recommendation: In a real-world enterprise environment, these assets should be downloaded locally, optimized/compressed, and served from a dedicated static images directory (e.g., /assets/images/) or via a reliable Content Delivery Network (CDN) to prevent dependency on third-party availability.

*Developed as a showcase of modern vanilla frontend capabilities.
